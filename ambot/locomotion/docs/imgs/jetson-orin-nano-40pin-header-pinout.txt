================================================================================
  PIN GROUPS BY FUNCTION
================================================================================

  POWER (4 pins):
    Pin  1 — 3.3V
    Pin  2 — 5V
    Pin  4 — 5V
    Pin 17 — 3.3V

  GROUND (8 pins):
    Pin  6, 9, 14, 20, 25, 30, 34, 39

  I2C (4 pins, 2 buses):
    Bus 7:  Pin  3 (SDA), Pin  5 (SCL)  — General purpose
    Bus 1:  Pin 27 (SDA), Pin 28 (SCL)  — EEPROM / ID bus

  UART (4 pins, 1 port):
    /dev/ttyTHS0:  Pin  8 (TX), Pin 10 (RX), Pin 11 (RTS), Pin 36 (CTS)

  SPI (10 pins, 2 buses):
    SPI0:  Pin 19 (MOSI), Pin 21 (MISO), Pin 23 (SCK), Pin 24 (CS0), Pin 26 (CS1)
    SPI1:  Pin 13 (SCK),  Pin 22 (MISO), Pin 37 (MOSI), Pin 18 (CS0), Pin 16 (CS1)

  I2S (4 pins, 1 bus):
    Pin 12 (SCLK), Pin 35 (FS), Pin 38 (SDIN), Pin 40 (SDOUT)

  PWM (3 pins):
    Pin 15 — GPIO12 / PWM1   (3280000.pwm channel 0)
    Pin 32 — GPIO07 / PWM7   (32e0000.pwm channel 0)
    Pin 33 — GPIO13 / PWM    (32c0000.pwm channel 0)

  GENERAL GPIO (4 pins):
    Pin  7 — GPIO09 (also Audio MCLK)
    Pin 29 — GPIO01
    Pin 31 — GPIO11
    (Most other pins can be reconfigured as GPIO via Jetson-IO / pinmux)

================================================================================
  NOTES FOR AMBOT PROJECT
================================================================================

  - For motor control (L298N/TB6612FNG/DRV8833), use PWM pins 15, 32, 33
    plus GPIO pins 7, 29, 31 for direction control.
  - I2C Bus 7 (pins 3/5) is best for IMU (MPU6050) — same role as RPi pins 3/5.
  - UART ttyTHS0 (pins 8/10) available for serial peripherals.
  - SPI buses available for additional sensors if needed.
  - Pin numbering: Use BOARD mode in Jetson.GPIO (same concept as RPi.GPIO).
  - Unlike RPi, the Jetson uses 3.3V logic — do NOT connect 5V signals directly.
  - All GPIO pins are 3.3V tolerant only (NOT 5V tolerant).
